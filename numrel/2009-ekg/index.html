3D interactive plots supplementing the article
Péter Csizmadia and István Rácz, Gravitational collapse and topology change in spherically symmetric dynamical systems
(submitted to Class. Quant. Grav.)

To see the plots, you need a 3D graphics card and a web browser which is able to run Sun's Java Plugin.

  • ℝ3 topology
    • Conserved energy (Kodama), apparent horizons — See Section 7.3.2, Fig. 11.
    • Kretschmann scalar, apparent horizon, equal τ hypersurfaces, light rays
  • S3 topology
    • Conserved energy (Kodama), apparent horizons — See Section 7.4, Fig. 14.
    • Kretschmann scalar, apparent horizons, equal τ hypersurfaces
  • S1×S2 topology
    • Conserved energy (Kodama) — See Section 7.5, Fig. 17.
    • Kretschmann scalar, apparent horizons, equal τ hypersurfaces

Troubleshooting — Read this section if the above linked pages do not work

  • Linux Java incompatibilities. The applet should work with Sun Java Plugin 1.6.0 update 14 and later versions but it does not work properly with most of the alternative Java implementations. The simplest solution is to install java-sun-plugin-* only and uninstall all packages with gcj, icedtea or openjdk appearing in their names.
  • "3D accelerated graphics is not available" error. Some possible solutions:
    • Try to enable 3D acceleration in your graphics card configuration.
    • Try to install a better driver for your graphics card.
    • If you have two graphics cards, then try to enable the more enhanced one. It can be done in the BIOS on some ThinkPads: choose Config → Display → Graphics Device → Discrete Graphics. Then 3D will work in Windows but not necessarily in Linux (without a proper driver). However, 3D acceleration may also work together with the Integrated Graphics setting in Linux.
